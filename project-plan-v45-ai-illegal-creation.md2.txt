# CreAIlity — Project Audit & Roadmap (v45)

## 1. WHAT EXISTS (Fully Functional)

### Core Builder Workspace — Production Quality
- React 19 + TypeScript + TailwindCSS v3 + Vite 8 SPA at `crealityapp.com`
- Auth: Supabase email/password with AuthGuard on `/workspace`, `/settings`, `/admin`
- AI engine (`ai-api.ts`): 5 models (GPT-4o, Claude 3.5 Sonnet, Gemini 2.0 Flash, DeepSeek V3, Grok 3), 3 build modes (Web App, React App, Browser Extension), direct client-side API calls using user's own API keys stored in localStorage
- Preview: sandboxed iframe via blob URL + Babel standalone bundler for React multi-file projects
- Persistence: localStorage (per-user scoped) + Supabase cloud sync (2s debounced push/pull with merge)
- Version history: snapshot on every build/save, restore/delete any version
- Import: GitHub repo (recursive tree fetch + batched file content), ZIP upload (drag-drop, JSZip extraction)
- Deploy: Vercel REST API + Netlify REST API using user's own tokens
- Full set of pages: Home, Features, Pricing, Docs, Contact, Terms, Privacy, Auth, Workspace, Settings, Admin, Preview

### Supabase Backend
- 4 tables: `projects`, `project_versions`, `messages`, `conversations` (latter two exist but unused in code)
- 5 edge functions: `stripe-setup`, `stripe-webhook`, `stripe-worker`, `AI Proxy`, `Site5 Deploy` (all exist on Supabase but NOT integrated with frontend)
- Auth fully wired, cloud sync fully wired

### Styling System
- OKLCH-based dynamic palette: 5 color roles (background, accent, primary, secondary, foreground) × 11 steps each
- Typography: Inter (body/label) + JetBrains Mono (code) from Google Fonts
- Dark mode support via `.dark` class CSS variable overrides

## 2. WHAT'S BROKEN / MISSING (Prioritized)

### CRITICAL — Security
- **RLS NOT ENABLED** on any of the 4 tables. Any authenticated user can technically query all rows. Apply immediately:
  - `CREATE POLICY "user_access" ON projects FOR ALL USING (auth.uid() = user_id);` (repeat for all 4 tables)
  - Admin bypass: `CREATE POLICY "admin_access" ON projects FOR ALL USING (is_admin());`
  - 15-minute fix, closes real vulnerability

### CRITICAL — Monetization (Zero Revenue)
- Pricing page is static HTML. "Start Pro" / "Start Team" buttons go to `/workspace` — no Stripe checkout
- `@stripe/react-stripe-js` and `@stripe/stripe-js` already in package.json but never imported
- `stripe-worker` edge function exists on Supabase — needs to be read, understood, wired to pricing page
- No plan enforcement: Free users get unlimited everything (all 5 models, unlimited projects, unlimited builds)
- No subscription tracking, no build counter, no gating anywhere

### HIGH — Mock/Fake Content (20 items)
All of these appear real to visitors but are static/fabricated: pricing CTAs, plan feature lists, stats numbers ("50k+ apps built"), 4 testimonials, 6 showcase apps, footer social links (all `href="#"`), navbar "Sign in" button (`href="#"`), FAQ team collaboration claim, integrations list (Stripe/Shopify/Cloudflare not actually integrated), settings "Plan" badge always shows "Free"

### HIGH — Publishing (User's Top Priority)
- Current preview uses blob URLs — die on page refresh. No persistent, shareable preview URL
- Cloudflare Sandboxes planned but not started (domain `crealityapp.com` is on Cloudflare DNS, wildcard record exists)
- Custom domain feature listed as Pro perk but zero implementation

### MEDIUM — Memory & Persistence
- Only last 10 conversation turns kept in AI context. Full history not stored
- No session resumption — every workspace open is a fresh start
- `conversations` and `messages` Supabase tables exist but never used

### MEDIUM — Admin Dashboard (30% Complete)
- Only 3 of ~8 needed tabs exist: Overview (4 stat cards), Users (table w/ search), Projects (table w/ search)
- Missing: Deployments, Edge Functions, Audit Log, System Settings, Billing Management, Analytics, User Detail, Project Detail

### MEDIUM — Auth Polish
- No email verification flow, no "forgot password" / password reset, no "resend verification" button

### MEDIUM — Build Reliability
- No build queue (spam clicks = simultaneous API calls), no rate limiting, no plan-based limits
- Raw API errors shown to users (no friendly messages), no auto-retry, no offline detection

### LOW — Missing Features
- Team collaboration (listed as Team plan feature — zero code exists)
- Browser extension .zip packaging for Chrome loading
- SEO: no structured data, sitemap, robots.txt, Open Graph tags, semantic HTML audit needed
- project_plan.md was completely outdated (this document replaces it)

### Style System Rules
- 5 color roles ONLY: background, accent, primary, secondary, foreground
- Each has 11-step scale (50-950). Use anchor tokens first: background-50, primary-500, accent-500, secondary-500
- Typography: `var(--font-heading)`, `var(--font-body)`, `var(--font-label)`, `var(--font-mono)`
- NEVER hardcode hex/rgb — always use token classes like `bg-primary-500 text-background-50`
